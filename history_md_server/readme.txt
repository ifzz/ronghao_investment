历史行情服务器的设计方式：
1、所有数据都以写文件的方式记录在服务器上

2、文件存储的目录结构为：history/国内期货/ni/16/ni1609/tick/20160602
第一个为根目录history，第二级分为上证、深证、国内期货等（A股根据交易所进行分类），第三级为品种，第四级为id中数字的
前两位，第五级为id，第六级分tick和diagram，tick表示深度行情，diagram表示指标数据，最后每天都建一个目录

3、在深度行情目录中的文件分为.tick和.ext两种类型的文件，其中的.ext是当日的整体性数据，包括当日开盘价、收盘价、
结算价、最高/低价等等，
DepthFileHead {
	char id[16];
	int version;
	int ms; //最小时间间隔，毫秒为单位
	char reserve[8]; 
}
紧接着是ext的结构

在.tick里面的也有一个头部结构，剩余的都是MarketDepthBase中的字段。.tick是每小时生成一个文件，对于期货来说，
间隔时间为500ms，所以文件最多有7200个base数据，对于指定时点上的数据来说，其在文件中的位置固定，
可以通过500ms和时间点快速定位到这一数据，假设在一个该来的时点上没有数据推送过来，仍然还是要写文件的。

在每个tick目录下都有一个day.list文件，这个文件最开始的4个字节表示有历史数据的最小日期，后面的是一个bytemap，
每个byte表示一个日期，从1900.1.1开始算起，如果在当前目录下有该日的历史数据，则在对应的byte上面将其值置为1,
总之需要通过时间可以迅速的定位到文件中的这个byte，判断这一日期是否有数据

A股的文件存储目录结构为：history/上交所/600/600135/diagram/2016/6/20160602
对于A股来说，在交易所下面再继续根据id的前三个数据分成三类股票，在这一目录下面才是指定的股票目录，
再下一级是diagram目录，在此目录下再根据年月日继续分，最后把文件放在最后的目录中，在年目录下（即2016）
有一个day.list文件，这一文件的功能和前述tick目录下的day.list文件相同

在底层目录下根据每种类型的指标数据都分成一系列的.data文件，比如kline_分钟_1.data文件。
