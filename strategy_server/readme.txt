生产环境和测试环境有许多重叠的功能，其中在收到合约列表、指标描述以及图表数据之后都需要做解析，并且在加载完策略之后，
策略的内存分配、读配置以及初始化，在触发买卖点时转发交易指令以及格式化地记录交易指令的功能也是这两个环境都需要共用的，
以上这两部分的功能都放在了ss_util模块中。

框架在初始化时主要完成以下操作：日志初始化、创建工作线程池、统计分析线程、在测试环境中还会创建接收子进程交易请求的线程
以及连接行情服务器。其中统计分析就是对当前进程占用的内存，cpu，磁盘I/O，打开文件等做一些记录，而工作线程池在两个环境中
分别完成不同的任务：①在生产环境中，线程池接收行情数据线程解析后的数据，并在这些工作线程上执行具体的策略；②在测试
环境中，行情数据线程只是简单转发接到的数据，在这些线程池中会将这些原始数据通过管道转发给所需的子进程，而解析和执行策略
